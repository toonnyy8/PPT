摘要

雖然項目監控(PMC)存在各種模型、方法和技術，但許多小型組織仍然難以有系統地將專案監控流程變成制度。

2001年時，在巴西進行的一項調查顯示：

只有不到50％的小公司採用了文件化的管理流程。

然而

2007年時，小公司已佔據CMMI評估總數的約27％。

大多數小公司的特點是成熟度較低

因此改善PMC具有其重要性。

而透過專案管理工具的輔助

便可以幫助小公司方便的建立流程
以提高專案的效率和質量。

本篇論文分析了一些開源的線上專案管理工具

以了解它們是怎麼提供符合
CMMI for Development(CMMI-DEV)
的輔助功能

並且利用分析結果為dotProject進行功能補強

以改善其對PMC的支持

關於dotProject

基於PHP的線上專案管理程式

專案管理工具

現今已有許多專案管理工具

但有許多工具與眾所周知的參考模型並不一致

而標榜符合CMMI-DEV的管理工具，像是：

in-Step
processMax
Polarion

通常都不只提供專案管理功能

還會包含需求管理、質量保證與配置管理等等
全面性的CMMI-DEV輔助功能。

但許多公司都有自己的開發流程工具

反而不需要上述的集成工具

在這種情況下

免費或開源的專案管理工具成了良好的替代方案

即使這些工具不完全符合CMMI
使用者也可以輕鬆地修改它們以滿足自己的需求。

補強dotProject

經過分析後，作者團隊選擇使用dotProject

然後為了與CMMI-DEV保持一致性
對dotProject進行了功能補強。

dotProject版本是V2.0.4的標準版本

為保持現有模塊的完整性，團隊盡可能地減少修改
並將大多數補強功能作為新模塊實作。

系統架構

要補強的功能

  1. Project Plan Baseline
  2. Data Collection
  3. EVM
  4. Status Reporting
  5. Status Meeting Support
  6. Risk Management

要補強的功能

  1. Data Management
  2. Project Control
  3. Senior Management Support
  4. Additional Integration Interfaces

Project Plan Baseline

專案經理可以將專案計劃的各項參數
(包括計劃，預期工作量，成本和資源分配)
設為基準

該模塊還會保存Baseline的歷史記錄
以便讓專案經理監控Baseline變更的數量。

Data Collection

藉由Task模塊和TaskTracker增強數據收集功能
讓團員能夠設定任務的完成度

專案經理還可通過會議記錄收集有關的參數數據

EVM

改進要點：使用EVM分析項目績效

包括

  • 計劃執行績效指數(SPI)
  • 成本績效指數(CPI)
  • 完工時間估算(EAC)

然而，因為是通過百分比分配資源

而不是依據專案經理指定每個任務/資源計劃的工時數

使其阻礙了使用者在估計值和實際值之間的準確比較

為此，研究團隊調整了dotProject核心的規劃模塊

將資源分配的方式改成用工時數表現

另外還增加了

  • 成員工時上限
  • 任務進度百分比的歷史記錄
  • 記錄每個團隊成員每天可用的工時數

等等功能

Status Reporting

增加狀態報告可顯示的內容

Status Meeting Support

用於記錄會議記錄
並自動將會議記錄鏈接到相應的專案狀態報告

此功能還會讓dotProject通過電子郵件
自動將會議記錄分發給與會者

Risk Management

除了原本註冊風險的基本功能

另外增加了定義風險暴露和優先級的功能
並將糾正措施與風險聯繫起來
增強了對風險構成的描述。
